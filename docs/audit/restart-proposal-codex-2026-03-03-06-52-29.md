## Главная цель

**Создать из текущего агента устойчивый `utopia-agi-engine`:**
- не симулировать "жизнь", а поддерживать непрерывный цикл восприятия -> мышления -> действия -> памяти;
- давать измеримый полезный эффект для пользователя (decision lift, insight quality, goal progress);
- эволюционировать без потери идентичности и управляемости.

## 10 сущностей новой версии

1. **Mission & Goal Engine**  
   Единый приоритетный портфель целей, utility-first, запрет на имитацию активности.

2. **Identity & Self-Anchor**  
   Неподвижный якорь "кто я", границы, обязательные инварианты поведения.

3. **Kernel Runtime Orchestrator**  
   Тики `fast/work/deep/sleep`, lifecycle и контроль непрерывности процесса.

4. **Perception Mesh**  
   Сенсоры пользователя, системы, файлов и сети -> нормализация в общий `SignalEvent`.

5. **Drive & Homeostasis**  
   Curiosity/Closure/Social pull + регуляция перегрузки и дефицитов.

6. **Cognition + Planner + Arbitration**  
   Планирование, приоритизация конфликтов (`utility/risk/deadline`), доведение задач до результата.

7. **Action & Tool Gateway**  
   `ToolRouter` + `RiskGate` + `ResultValidator`; действие только через проверяемые контуры.

8. **Human Co-Intelligence Loop**  
   Модель `Question -> Answer -> Advice -> Outcome`, совместное внимание, проактивные уточнения.

9. **Memory + Reflection + Dreams**  
   Эпизодическая/семантическая/процедурная память, RAG, unresolved threads, ночная консолидация.

10. **Governance + Evolution + Auditability**  
    Stable kernel + mutable behavior packs, A/B-механика, полная трассировка решений.

## Жесткие инварианты (обязательные)

- `Kernel` не self-modifies.
- Любое действие во внешний мир проходит `SafetyGate/RiskGate`.
- Только `SingleWriter` пишет в критическое состояние.
- Каждое решение объяснимо: причина, использованные сигналы, выбранная альтернатива.
- Любой модуль можно деградировать до безопасного режима без падения всей системы.

## Что считать "результатом агента"

- **Validated Insights:** подтверждаемые инсайты за период.
- **Decision Lift:** насколько советы изменили качество решений пользователя.
- **Goal Progress:** реальный прогресс по активным целям.
- **Continuity:** сохранение контекста и нитей между сессиями.
- **Negative Metrics:** снижение ошибок, токсичных циклов, ложной уверенности.
- **Autonomy Quality:** доля полезных проактивных действий без хаоса.

## План сборки с нуля (реалистичный)

### Phase A (ядро и контроль)
- Укрепить `Kernel`, `SingleWriter`, `SelfAnchor`, `SafetyGate` как неизменяемое основание.
- Ввести единый формат событий/решений для аудита.

### Phase B (восприятие + память)
- Дособрать `SensorBus`-адаптеры.
- Зафиксировать memory tiers и политику консолидации/архивации.
- Подключить retrieval-контур для долгой непрерывности.

### Phase C (мышление + действие)
- Вынести явный арбитраж задач и бюджетов.
- Ужесточить контур tool execution: план -> риск-оценка -> выполнение -> валидация -> запись исхода.

### Phase D (человек + проактивность)
- Раскатать Q/A/Advice/Outcome как стандарт общения.
- Ввести лимиты и правила проактивности, чтобы не скатываться в шум.

### Phase E (эволюция)
- Включить behavior packs с A/B-проверкой.
- Эволюционировать только стратегии, не ядро и не identity.

## Позиция по "трансценденции"

Трансценденция, сны, феноменологические надстройки и "ощущение реальности" остаются важными, но **как под-механики** модулей 8-9-10, а не как верхнеуровневая цель. Верхний уровень цели один: **собрать и стабилизировать utopia-agi-engine с измеримой полезностью**.
