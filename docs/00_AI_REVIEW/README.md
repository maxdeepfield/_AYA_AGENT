## На Что Смотреть

Другой ИИ должен выбирать не самый красивый README, а самый перспективный прототип по таким критериям:

1. Есть ли у системы собственное текущее дело, а не только чат.
2. Двигается ли она действиями, а не только narration.
3. Не сваливается ли она в ритуальный loop.
4. Есть ли у неё структура, которую реально можно развивать.
5. Не убивает ли архитектура базовую умность модели.
6. Есть ли в проекте что-то уникальное, что не стоит выбрасывать.

## Практический Совет

Пусть внешний ИИ не пытается сразу читать всё подряд.
Лучше:

1. пройтись по prototypes.json
2. отобрать 3-5 самых интересных кандидатов
3. для них прочитать README, SPEC, docs и поверхностно посмотреть src
4. выдать сравнение, что сохранить, что выкинуть и на чём строить дальше
