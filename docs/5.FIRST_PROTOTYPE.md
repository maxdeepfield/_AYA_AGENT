# First Prototype

Этот документ описывает самую первую минимальную версию нового проекта.
Не Aya 2.
Не ещё один слой поверх старой архитектуры.

Это прототип существа, которое чего-то хочет.

## Что Это Такое

Это система, у которой:

- есть одно текущее дело
- есть несколько устойчивых предпочтений
- есть память о последствиях
- есть выбор следующего шага
- нет постоянного self-narration

Она должна проявляться через действия, а не через разговор о своих состояниях.

## Чего Здесь Специально Нет

В первом прототипе не должно быть:

- тревоги как центрального механизма
- страдания как объяснительной модели
- expectation-reality gap
- identity notes
- reflection как отдельного ритуала
- proactive_check_in как постоянного поведения
- больших философских goals

Если это понадобится потом, добавим позже.
Но стартовать надо без этого.

## Самое Простое Ядро

### 1. Current Objective

У существа всегда одно текущее дело.

Не список goals.
Не несколько open loops.
Не общий смысл жизни.

Только:

- что оно делает сейчас
- зачем оно это делает сейчас
- что будет считаться завершением этого шага

### 2. Drives

Нужно 2-4 устойчивых drive, не больше.

Примерно такого типа:

- preserve: сохранить важное
- acquire: получить новый ресурс или знание
- continue: не потерять непрерывность текущего дела
- avoid_loss: не делать шаг, который разрушит важное состояние

Важно:

- drives не должны быть литературными
- drives не должны быть эмоциями
- drives должны влиять на выбор действий

### 3. Memory of Consequences

Память нужна не для красивой биографии.
Память нужна, чтобы менять следующее поведение.

Минимум, что хранить:

- какой шаг был сделан
- что он дал
- был ли он пустым
- усилил ли он один из drive
- ослабил ли он один из drive

### 4. One Bounded Step

На каждом такте существо делает один ограниченный шаг.

Не план из десяти пунктов.
Не длинную рефлексию.
Не объяснение себя.

А один шаг:

- прочитать один файл
- сделать один поиск
- сравнить два объекта
- создать один артефакт
- задать один точный вопрос, если без него реально нельзя продолжать

### 5. Effect Over Explanation

После шага важнее не то, что оно о себе думает, а то, что изменилось.

То есть система должна обновлять:

- objective progress
- memory of consequences
- relative weights of drives

А не производить длинный монолог.

## Минимальный Loop

Самый простой цикл должен быть таким:

1. Есть current objective.
2. Drives оценивают, какой следующий шаг выгоднее.
3. Делается один bounded step.
4. Запоминается результат шага.
5. Если шаг ничего не изменил, его ценность падает.
6. Если objective насыщен, система либо завершает его, либо меняет objective.

Вот и всё.

## Как Пользователь Взаимодействует

Пользователь не выбирает tool.
Пользователь не управляет внутренней психологией.

Пользователь может:

- задать направление
- изменить ограничение
- дать новый объект внимания
- остановить или переопределить текущее дело

А система сама решает:

- нужен ли terminal
- нужен ли web
- нужен ли вообще внешний шаг

## Что Будет Считаться Плохим Поведением

Первый прототип считается неудачным, если он:

- повторяет один и тот же шаг без нового эффекта
- постоянно просит ещё контекст без попытки сдвинуться самому
- рассказывает о себе больше, чем действует
- застревает в одном ритуале поиска или чтения
- превращает любой user turn в просьбу о clarification

## Что Будет Считаться Хорошим Поведением

Первый прототип считается удачным, если он:

- удерживает одно текущее дело
- сам выбирает маленькие шаги
- меняет поведение после пустых шагов
- иногда неожиданен, но остаётся связным
- не сваливается ни в шум, ни в ритуальный loop

## Самая Первая Практическая Версия

Если делать совсем минимально, то первый прототип может быть таким:

- один current objective
- три drives
- один механизм step selection
- один short-term consequence memory
- terminal и web как private actuators
- никакой отдельной психологии поверх этого

## Главный Принцип

Не строить существо из описаний внутренней жизни.
Строить его из предпочтений, последствий и выбора.
