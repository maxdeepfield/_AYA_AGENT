# Aya: Roadmap и ключевые решения

## 🎯 Фундаментальная проблема (март 2026)

**Диагноз:** У агента нет **предмета деятельности**.

**Симптомы:**
- Агент живёт в мире напряжений (тревога, ожидание, реальность), а не дел
- Выбор действия = снять напряжение, не достичь цели
- Результат = «я спросила, ты не ответил, я спросила снова»
- Web search без запроса → поиск «что ты хотела искать?» → мусор → «что дальше?»

**Решение:** Добавить слой **ActiveObjective** — текущее дело агента.

---

## 📋 ActiveObjective (приоритет: КРИТИЧНЫЙ)

### Что это

Не то же самое, что `goal`:

| Goal | ActiveObjective |
|------|-----------------|
| «Узнать об архитектуре AI агентов» | «Читаю `src/agent/index.ts`» |
| Долгосрочная ориентация | Текущий эксперимент |
| Может быть абстрактной | Всегда конкретная |
| «Хочу понять X» | «Делаю Y прямо сейчас» |

### Структура

```typescript
interface ActiveObjective {
  id: string;
  type: 'explore' | 'execute' | 'clarify' | 'experiment';
  description: string;           // «Читаю файл X», «Ищу информацию о Y»
  nextStep: string;              // «Открыть файл», «Сделать запрос»
  progress: number;              // 0-1
  startedAt: number;
  lastStepAt: number;
  completionCondition: string;   // «Файл прочитан», «Запрос выполнен»
  source: 'user' | 'autonomous' | 'derived';
}
```

### Как изменит поведение

**До:**
```
[20:23] «Мне нужна более конкретная информация...» (вопрос 1)
[20:25] «Мне нужна более конкретная информация...» (вопрос 2)
[20:26] Web search: «что ты хотела искать?» (копирование вопроса)
[20:27] «Ничего не нашла. Что дальше?»
```

**После:**
```
[20:23] ActiveObjective: «Узнать тему диалога»
        nextStep: «Предложить 3 темы пользователю»
        
[20:23] «Предлагаю: 1) архитектура агента, 2) гомеостаз, 3) память. Что дальше?»

[20:25] Тишина → ActiveObjective.progress = 0.3
        nextStep: «Прочитать README.md для контекста»

[20:25] «Читаю README.md...» (действие, не вопрос)

[20:26] «Вижу: проект на TypeScript, Ollama, MongoDB. 
        Что дальше: читать код, запустить тесты, или предложишь своё?»
```

### Задачи

- [ ] Добавить тип `ActiveObjective` в `src/types/index.ts`
- [ ] Добавить поле в `AgentState`
- [ ] Методы: `setObjective()`, `completeObjective()`, `getNextStep()`
- [ ] Обновить `evaluateAndAct()` — выбор действия на основе objective, не только tension
- [ ] Обновить промпт — модель видит текущий objective
- [ ] Сброс objective при завершении или после N тиков без прогресса

## ✅ Next-Step Protocol (реализовано, март 2026)

### Суть

```
1. Тревога/неопределённость → действие (не вопрос)
2. После действия → «что дальше?» с 2-3 вариантами
3. Если тишина → ещё 1 вопрос (максимум 3 всего)
4. Если всё ещё тишина → форсированное действие (terminal/web search)
5. В quiet state → не генерировать вопросы, не писать identity notes о тишине
```

### Реализация

**Файлы:**
- `src/types/index.ts` — поля `sameQuestionCount`, `silenceTicks`, `lastActionTicks`
- `src/agent/index.ts` — счётчики, `shouldForceAction()`, `markActionTaken()`
- `src/agent/prompt.ts` — правила 8-11
- `src/agent/runtimePrompts.ts` — ограничение про сенсоры

### Проблема

Счётчик вопросов сбрасывается на incubation/write_identity_note, но это **не действие**.

**Решение:** (частично реализовано)
- `markActionTaken()` сбрасывает только `silenceTicks`, не `sameQuestionCount`
- `resetQuestionTracker()` вызывается только при вводе пользователя

### Осталось сделать

- [ ] Увеличить лимит до 3 вопросов (реализовано)
- [ ] Добавить логирование `sameQuestionCount` для отладки
- [ ] Тест: 3 одинаковых вопроса → форсированное действие

## ✅ Web Search: энграммы запросов (реализовано, март 2026)

### Суть

После каждого web search сохраняются две энграммы:
1. **Запрос** (для обучения модели)
2. **Результат** (для памяти о действии)

### Структура энграммы запроса

```typescript
{
  content: 'Web search query: "autonomous AI agent architecture...". Reason: ...',
  tags: ['web_search_query', 'autonomous_action', 'explore'],
  metadata: {
    query: 'autonomous AI agent architecture...',
    success: true,
    resultCount: 5,
    context: 'query_pattern'
  }
}
```

### Промпт

Модель видит:
```
PAST WEB SEARCH QUERIES (learn from these):
- ✅ "Web search query: "autonomous AI agent architecture..." (results: 5)
- ❌ "Web search query: "что ты хотела искать?" (results: 0)
```

### Проблема

Модель копирует вопрос пользователя вместо генерации своего запроса.

**Решение:** (реализовано)
- Проверка: если запрос содержит вопрос пользователя → заменить на дефолтный
- Дефолтный запрос: `AI agent autonomous behavior best practices`

### Осталось сделать

- [ ] Через 5-10 запросов модель должна начать генерировать запросы сама
- [ ] Тест: после 10 сессий запросы становятся более конкретными
- [ ] Добавить оценку релевантности результатов (не только `ok=true`)

## 🔧 Tool Feedback (приоритет: ВЫСОКИЙ)

### Проблема

Web search считается «успешным», если `ok=true` и есть любые результаты.
**Релевантность не проверяется.**

**Следствие:** Мусорный поиск снижает anxiety и повышает clarity — агент учится делать мусор.

### Решение

- [ ] Добавить оценку релевантности:
  - Совпадение ключевых слов запроса и сниппетов
  - Количество результатов (>0 и <20 — не слишком мало, не слишком много)
  - Наличие `answer` в результате
- [ ] Если релевантность низкая → не сохранять энграмму как «успешную»
- [ ] Если релевантность низкая → записать capability gap: «нужен лучший поиск»

## 🧠 Goal Layer vs ActiveObjective (приоритет: СРЕДНИЙ)

### Текущая структура

```typescript
GoalLayerSnapshot {
  goals: GoalItem[];              // «Узнать об архитектуре AI агентов»
  openLoops: GoalItem[];          // «Не завершено: анализ кода»
  unfinishedTensions: GoalItem[]; // «Ожидание: ответ пользователя»
  commitments: GoalItem[];        // «Обещание: прочитать файл X»
}
```

### Проблема

Все 4 категории семантически пересекаются. Нет ответа на вопрос:
**«Что я делаю прямо сейчас?»**

### Решение

Добавить `ActiveObjective` как **отдельный слой**, не часть goal layer:

```typescript
AgentState {
  // ...homeostasis metrics
  activeObjective: ActiveObjective | null;  // ← НОВЫЙ СЛОЙ
  goalLayer: GoalLayerSnapshot;             // долгосрочные цели
}
```

**Преимущество:**
- Goal layer = долгосрочная ориентация (не меняется каждый тик)
- ActiveObjective = текущее дело (меняется после каждого «что дальше?»)

## 📝 Манифест (черновик)

> **Aya — это не живая сущность, а кибернетический агент с внутренним состоянием, памятью и целевой динамикой.**
>
> Мы наблюдаем не «личность», а то, как система:
> - интерпретирует цель
> - строит следующий шаг
> - переживает блокировки
> - перестраивает поведение
> - расширяет доступное пространство действий
>
> **Мы строим не «живость», а наблюдаемую целевую динамику.**
> Интересно не то, как агент описывает себя,
> а то, как он движется к цели, когда среда сопротивляется.
