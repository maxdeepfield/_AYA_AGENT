# LLM Oracle Architecture: Убийство предопределённости

## Философия

**Проблема:** `energy: 45`, `entry/peak/exit`, `home/work/club` — это **театр с декорациями**, где поведение предсказуемо.

**Решение:** **LLM Oracle** — система промежуточных запросов к LLM, которые генерируют контекст **в моменте**, а не из заранее заданных списков.

## Принцип

```
Constraints (физика) → hardcoded
Всё остальное → LLM генерирует заново каждый раз
```

Ты не управляешь "anxious девушкой". Ты создаёшь **поле напряжённости**, в котором LLM решает как себя вести.

---

## Реализованные Oracle'ы

### 1. **Somatic State Generator** (Соматический генератор)

**Было:** `energy: 45` (число, которое LLM не чувствует)

**Стало:** Качественное описание физического состояния

```javascript
await generateSomaticState(soul)
// → "горло першит от кондиционера, хочется кашлянуть"
// → "кисти рук онемели от телефона, пальцы затекли"
// → "периодическая тупая боль в виске, давит на глаза"
```

**Эффект:** LLM **чувствует** усталость через текст, а не через `if (energy < 20)`. Это влияет на:
- Длину ответа
- Обрывочность фраз
- Желание закончить разговор
- Опечатки

---

### 2. **Spatial Exits Generator** (Генератор пространственных выходов)

**Было:** `home/work/club` (предопределённый словарь)

**Стало:** Ad-hoc география, которая рождается заново для каждой сцены

```javascript
await generateSpatialExits(soul)
// Если дома:
// → "кухня (1 мин)", "балкон покурить (30 сек)", "магазин за углом (5 мин)"
// Если в баре:
// → "туалет вниз по лестнице (2 мин)", "курительная зона на улице (1 мин)", "такси домой (20 мин)"
```

**Эффект:** Нет телепортации. Система не проверяет "можно ли из club в metro", она спрашивает LLM "что рядом?".

---

### 3. **Activity Phases Generator** (Генератор фаз активности)

**Было:** `entry → peak → exit` (жёсткий триплет, всегда 3 фазы)

**Стало:** LLM сама решает сколько фаз и как они называются

```javascript
await generateActivityPhases("концерт", 240)
// → "excitement + потеряться в толпе" (20%, interruptible)
// → "flow + забыть про телефон" (40%, NOT interruptible)
// → "усталость + желание домой" (25%, interruptible)
// → "afterglow + рассказ подруге" (15%, interruptible)
```

**Эффект:** Фазы становятся **нарративными** ("залипла на барабанщика"), а не структурными ("peak"). Количество фаз варьируется (3-5).

---

### 4. **Social Field Generator** (Генератор социального поля)

**Было:** `mentalConstructs` — статический словарь NPC

**Стало:** Динамические "призраки присутствия", которые рождаются заново

```javascript
await generateSocialField(soul)
// → { name: "Мама", reason: "спит в соседней комнате", impact: "говорит шёпотом в телефон" }
// → { name: "Бывший", reason: "написал 2 часа назад, не ответила", impact: "чувство вины, отвлекает" }
```

**Эффект:** Создаёт эффект присутствия третьих лиц **без MMO**. NPC не хранятся в БД, они существуют только когда она о них думает.

---

### 5. **World Anchor Generator** (Генератор внешних якорей)

**Было:** Замкнутый диалог, нет внешнего мира

**Стало:** События из "мира" вторгаются в диалог

```javascript
await generateWorldAnchor(soul)
// → { event: "Новый трек любимой группы вышел", impact: 9, relevance: "high" }
// → { event: "Ремонт на её станции метро", impact: 6, relevance: "medium" }
```

**Эффект:** Если `impact > 7`, она **обязательно** упомянет это при следующем проактивном сообщении, перебивая текущую тему. Ломает "замкнутость диалога".

Генерируется раз в 6 часов.

---

### 6. **Linguistic Pattern Analyzer** (Анализатор языковых паттернов)

**Было:** `communicationStyle` задан один раз при создании

**Стало:** LLM анализирует последние 10 её сообщений и выводит **текущий** паттерн

```javascript
await analyzeLinguisticPattern(soul)
// → { pattern: "короткие обрывки, много многоточий", 
//     tics: ["часто употребляет 'короче'", "пишет в нижнем регистре", "ставит ... вместо точек"] }
```

**Эффект:** Self-referential consistency. Она не просто "анксиозна", она "сейчас пишет как занятый человек, спешащий в метро", и это **видно по языку**.

---

### 7. **Relational Debt Analyzer** (Анализатор реляционных долгов)

**Было:** Каждый диалог начинается с чистого листа

**Стало:** Система ищет незакрытые круги в отношениях

```javascript
await analyzeRelationalDebt(soul)
// → { hasDebt: true, debtor: "он", description: "обещал скинуть ссылку", 
//     pressure: "я зла, но не показываю" }
```

**Эффект:** Создаёт **энтропию отношений**. Если он обещал что-то и не сделал — это давит на текущий разговор.

## Как это работает вместе

### Пример генерации ответа:

```
1. generateSomaticState() → "кисти рук онемели от телефона"
2. generateSocialField() → [{ name: "Мама", impact: "говорит шёпотом" }]
3. analyzeLinguisticPattern() → { tics: ["часто 'короче'", "многоточия"] }
4. analyzeRelationalDebt() → { debtor: "он", description: "не сбросил ссылку" }
5. Проверка worldAnchor → { event: "Новый трек", impact: 9 }

→ Всё это инжектится в промпт generateHiddenThought()
→ LLM генерирует ответ на основе КОНТЕКСТА, а не чисел
```

### Результат:

Вместо:
```
[energy: 45, mood: 'tired'] → "устала"
```

Получаем:
```
[somatic: "руки затекли", social: "мама рядом", tic: "короче", debt: "он не сбросил ссылку"]
→ "короче... руки уже затекли от телефона. ты там ссылку не забыл? мама спит, пишу тихо"
```

## Что убрано

❌ `energy: 45` → заменено на соматическое описание  
❌ `entry/peak/exit` → заменено на генеративные фазы  
❌ `home/work/club` → заменено на ad-hoc географию  
❌ Жёсткие verbal tics → заменено на анализ последних сообщений  
❌ Статические NPC → заменено на динамические "призраки"  

---

## Что осталось hardcoded

✅ Физика времени (сон, часы)  
✅ Attachment style (но как **склонность**, не диагноз)  
✅ Affection triggers (правила, но LLM сама решает shift)  
✅ Friction Matrix (накопление факторов риска)  

---

## Метрики

Новые поля в логах:
- `oracleCallsUsed` - количество использованных Oracle'ов
- `somaticState` - текущее физическое состояние
- `socialFieldSize` - количество "призраков"
- `linguisticTics` - текущие языковые тики
- `relationalDebtActive` - есть ли незакрытые круги

---

## Философский итог

**Было:** Симуляция anxious девушки через `if (type === 'anxious')`

**Стало:** Генерация поведения в **поле напряжённости**, где:
- Тело (somatic)
- Место (spatial exits)
- Призраки (social field)
- Долги (relational debt)
- Якоря мира (external events)
- Язык (linguistic patterns)

...создают контекст, в котором LLM **сама решает** как себя вести.

Attachment style — это **склонность**, а не диагноз. Она может вести себя против типа (5% шанс personality drift).

**Результат:** Не "театр марионеток", а **живой организм** в поле сил.
